# PharmaGuard – Pharmacogenomic Risk Prediction System

PharmaGuard is a full‑stack web application that analyzes patient VCF (Variant Call Format) files to predict drug‑gene interactions and provide clinical recommendations based on CPIC guidelines. It combines a React + TypeScript frontend with an Express + MongoDB backend, and uses **Groq's Llama 3.3 70B** model to generate patient‑friendly explanations.

## Features

- 📤 **VCF Upload** – Drag‑and‑drop or select a `.vcf` file (max 5 MB).
- 💊 **Drug Selection** – Multi‑select from a list of pharmacogenomically relevant drugs (Codeine, Warfarin, Clopidogrel, etc.).
- 🧬 **Variant Matching** – Matches rsIDs from the VCF against a curated MongoDB collection of pharmacogenetic variants.
- ⚖️ **Risk Assessment** – Determines phenotype, risk level (Safe / Adjust Dosage / Toxic‑Ineffective), and confidence score.
- 📋 **CPIC Recommendations** – Provides dosage adjustments, alternative drugs, and monitoring notes based on CPIC guidelines.
- 🤖 **AI Explanations** – Uses Groq's **Llama 3.3 70B** model to generate plain‑language summaries of the results.
- 📊 **Detailed Results** – Six‑tab display: Risk Summary, Genomic Profile, Recommendation, AI Explanation, Quality Metrics, and JSON output.
- 📜 **History & Dashboard** – All analyses are saved in `localStorage` and can be reviewed on the History page; the Dashboard shows real‑time statistics.

The application will be available at http://localhost:8080 (or the port shown by Vite).

Usage Guide
Navigate to the Analysis page (sidebar or click “Start New Analysis” on the Dashboard).

Upload a VCF file – drag & drop or click to select a .vcf file (max 5 MB). You can download a sample VCF using the button provided.

Select one or more drugs – use the dropdown or enter comma‑separated drug names.

Click “Run Analysis” – the backend will parse the VCF, match variants, and return a detailed report.

Explore the results – use the tabs to view:

Risk Summary – overall risk, severity, and reasoning.

Genomic Profile – primary gene, diplotype, phenotype, and detected variants.

Recommendation – CPIC guideline‑based advice.

AI Explanation – plain‑language summary generated by Groq's Llama 3.3 70B.

Quality Metrics – parsing statistics and any warnings.

JSON Output – raw JSON data for download or copy.

Review history – all analyses are saved locally and appear on the History page and Dashboard.

API Documentation
POST /api/analyze
Accepts a VCF file and a list of drugs, returns an array of risk assessments (one per drug).

Request (multipart/form‑data):

vcf_file – the .vcf file

drugs – JSON string array, e.g. ["CODEINE","WARFARIN"]

Response:

json
{
  "success": true,
  "data": [
    {
      "drug": "CODEINE",
      "patient_id": "PATIENT_123456",
      "timestamp": "2025-02-20T12:00:00.000Z",
      "risk_assessment": { ... },
      "pharmacogenomic_profile": { ... },
      "clinical_recommendation": { ... },
      "llm_generated_explanation": { ... },
      "quality_metrics": { ... }
    }
  ]
}
GET /api/health
Simple health check.

Response:

json
{ "status": "OK", "message": "Server is running" }
Deployment
Frontend (Vercel)
Push your code to a GitHub repository.

Import the project on Vercel.

Set the root directory to client (if your repo contains both frontend and backend).

Add the environment variable VITE_API_URL pointing to your deployed backend URL (e.g. https://your-backend.onrender.com/api).

Backend (Render)
Create a new Web Service on Render.

Connect your repository and set the root directory to backend.

Add the environment variables (MONGO_URI, GROQ_API_KEY, etc.).

Set the start command: npm start.

Database (MongoDB Atlas)
Create a free cluster on MongoDB Atlas.

Obtain your connection string and add it to the backend’s MONGO_URI.

Customization
Add more drugs/variants – Update SUPPORTED_DRUGS in Analysis.tsx and seed the database with corresponding rsIDs in variantSeed.json.

Modify phenotype logic – Edit phenotypeEngine.js to include additional gene‑diplotype rules.

Change risk mapping – Update mapRisk in analyzeController.js to adjust how risk strings are converted to labels/severities.

Switch AI model – In llmService.js, change the model name in the Groq API call (see Groq documentation).

Troubleshooting
Error	Likely Cause	Solution
No known pharmacogenetic variants found	Database not seeded or VCF contains no matching rsIDs	Run npm run seed in backend; check that rsIDs in VCF exist in seed data.
CORS error: Access-Control-Allow-Origin mismatch	Frontend origin not allowed	Ensure cors() in server.js allows your frontend URL (or use cors() without options).
Groq API 401 or 429	Invalid API key or rate limit exceeded	Verify key in .env; check usage at Groq Console.
Cannot read properties of undefined (reading 'risk_label')	Frontend expects different data shape	Check that backend returns the structure defined in AnalysisResult type. Use the controller code from earlier answers.
